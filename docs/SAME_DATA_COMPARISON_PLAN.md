# Same-Data Comparison Plan

## Why Same Metric Is Not Enough

Using the same metric as a prior paper does not create a direct comparison by itself. Path F1, edit distance, NDTW, Hit@K, or MRR can change substantially depending on how trips are filtered, how the road graph is built, how GPS traces are map-matched, how origin-destination queries are generated, and what candidate paths are available.

Two methods can report the same metric name while solving different tasks. For example, one system may learn the search process and generate paths directly, while another may rerank a small candidate set generated by OSM shortest-path variants. Those results are informative but not directly comparable unless the task setup is matched.

## Why Same Dataset and Task Setup Are Needed

A direct comparison requires the same dataset and the same task definition. The following must be matched before claiming that one method beats another:

- Dataset.
- Road graph and preprocessing.
- Query generation.
- Train/test split.
- Candidate set or search method.
- Metrics and thresholds.

Without these controls, a performance difference may reflect preprocessing or candidate-pool difficulty rather than the ranking or search model.

## 2024 User-Habit Paper

The 2024 user-habit route recommendation paper is conceptually close because it studies route sorting from historical navigation behavior. It is not directly comparable to this project yet because it relies on private navigation logs and route-choice labels. Our current public data does not contain equivalent persistent user histories or explicit choice labels.

The paper should be cited as motivation for profile-based route sorting, not as a beaten baseline.

## Why Porto Is the Public Benchmark Path

The Porto taxi trajectory data is a public ECML/PKDD 2015 benchmark dataset with trajectories from 442 taxis in Porto, Portugal. It gives us a recognized public trajectory source for same-data experiments.

Porto is stronger than the exploratory Toronto OSM trace probe for benchmarking because it is public, known, and vehicle-route oriented. The current Porto setup reconstructs observed OSM driving routes, generates candidates for the same origin and destination, and evaluates ranking plus path-overlap metrics.

## Why NASR Is the Future External Baseline Target

NASR is the strongest future external baseline target because it learns neural cost functions inside A* search for personalized route recommendation. It is a route-generation/search method, not only a candidate reranker.

Our current system reranks candidates generated by OSM graph search. A direct NASR comparison requires reproducing NASR preprocessing, splits, graph construction, and path-overlap metrics on the same dataset.

## Current Comparison Status

- Internal same-data comparison on Porto: valid.
- Direct NASR comparison: pending reproduction.
- Direct comparison to the 2024 user-habit paper: not valid unless equivalent data becomes available.

## Current Safe Claim

We present an interpretable OSM-based route-candidate reranking framework and show that trajectory-derived profiles can improve ranking metrics on a public Porto benchmark, while reconstruction/path metrics expose remaining limitations.
