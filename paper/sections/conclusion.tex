This paper presented a dynamic OSM-based route-ranking system that builds pseudo-history profiles from public GPS-derived movement signals, extracts interpretable OSM route features, and ranks generated route candidates using profile, prompt, or hybrid modes. The main methodological point is separation: reconstruction quality, candidate-generation quality, and ranking quality are evaluated independently so that noisy public traces are not mistaken for clean user histories.

The current results show both promise and limits. In the OSM public-trace setting, the pipeline produces usable exploratory pseudo-history signals but fails strict reconstruction usability because route-distance ratio remains high. In the Porto public-dataset benchmark, a vehicle-aware profile ranker improves over shortest-distance and a generic profile on several ranking metrics, while path metrics show that map matching and candidate generation remain bottlenecks. The result is a submission-ready research framework: transparent, reproducible, and honest about what is proven. The next step is a larger Porto benchmark with a reproduced external baseline and stronger map matching, which would allow a direct claim about whether interpretable profile-aware reranking can compete with learned personalized route-search methods.
