This draft presented a prototype for dynamic OSM-based profile routing from public GPS trace signals. The system uses OSM-derived historical movement signals to construct pseudo-history profiles, extracts interpretable OSM route features, and ranks generated walking-route candidates using profile, prompt, or hybrid modes. The current evidence supports an exploratory research direction but not yet a final claim: trace timestamp coverage and GPS-to-route proximity are strong, but strict reconstruction usability fails because route-distance ratio remains too high.

The immediate next steps are to improve trace reconstruction quality, expand the evaluation set, add preference text or a clean user-history source for prompt/hybrid evaluation, and compare against stronger learned baselines. With those improvements, the project can move from a working demo toward a research-defensible route recommendation study.
