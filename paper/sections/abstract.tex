Personalized route recommendation is usually evaluated with clean navigation logs, explicit user histories, or proprietary trajectory data. This paper advances an open-data alternative: preference-aware route ranking from OpenStreetMap (OSM) features and public GPS-derived movement signals. We present a working route-ranking system that generates OSM route candidates, extracts interpretable route features, constructs contextual pseudo-history profiles, and reranks candidates using profile, prompt, or hybrid scoring. The paper makes validity an explicit part of the method by evaluating trace reconstruction and route ranking separately, rather than treating public GPS traces as unquestioned user histories. On an OSM public-trace probe, timestamp coverage and GPS-to-route proximity are strong, while route-distance ratio identifies where reconstruction quality must still improve. To align with recognized public trajectory benchmarks, we add a Porto taxi trajectory evaluation with OSM driving-route candidates, feature/path oracles, bootstrap confidence intervals, and NASR-style path-overlap metrics. In this 50-query Porto benchmark, a vehicle-aware profile ranker improves over shortest-distance and a generic profile on Hit@1, Hit@3, MRR, NDCG@3, and mean feature distance. Path-level metrics further reveal how map matching and candidate generation shape the achievable ceiling. The result is a transparent, reproducible framework for interpretable, preference-aware OSM route ranking under realistic open-data constraints.
