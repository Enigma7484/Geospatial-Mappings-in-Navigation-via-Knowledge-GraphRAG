Personalized route recommendation usually assumes access to clean user histories, explicit preferences, or production navigation logs. This paper drafts a research prototype that instead studies whether OpenStreetMap (OSM) public GPS trace signals can support exploratory preference-aware route ranking. The system constructs OSM-derived pseudo-history profiles from public GPS trackpoints, extracts route-level features from OSM walking graphs, builds dynamic profile weights conditioned on temporal context, and ranks generated route candidates using profile-based, prompt-based, or hybrid scoring. Current results should be read as prototype evidence, not final claims: timestamp coverage and GPS-to-route proximity are strong in the Toronto probe, while strict reconstruction usability fails because the median route-distance ratio exceeds the target range. A route-candidate baseline evaluation shows that profile ranking is competitive with shortest-distance ranking under feature-similarity relevance, but the dataset is too small for broad conclusions. The paper scaffold defines the method, baselines, limitations, and next steps needed to make the system research-defensible.
